# Hands-On Cybersecurity Lab – System Hardening with Kali Linux & Windows Server

## Project Overview
This project focused on implementing and understanding core **system hardening** techniques used to improve the security posture of enterprise systems. The lab was conducted using **Kali Linux** and **Windows Server**, with the objective of applying proactive security measures to reduce vulnerabilities and strengthen system defenses.

The project demonstrates practical experience in:
- Patch management and system updates
- Firewall configuration and network security
- File permissions and access control
- Service and device management
- Defensive security and system hardening principles

![System Hardening Lab](./Screenshots/lab-overview.png)

This lab simulates foundational Blue Team activities commonly performed by system administrators, SOC analysts, and cybersecurity professionals.

---

## Tools and Technologies Used
- **Kali Linux**  
  Security-focused Linux distribution used for cybersecurity testing and administration.

- **Windows Server**  
  Enterprise operating system used to configure and manage security settings.

- **Windows Defender Firewall**  
  Used to create and manage inbound and outbound firewall rules.

- **Command Line / PowerShell**  
  Used for system administration, service management, and configuration tasks.

- **Virtualization Environment**  
  Used to safely conduct testing in an isolated lab environment.

---

## Step-by-Step Setup and Execution

### 1. Lab Environment Setup
- Installed and configured Kali Linux and Windows Server virtual machines
- Verified connectivity between systems
- Created an isolated lab environment for safe testing

### 2. Installing Security Updates and Patches
- Checked for missing updates and installed available security patches
- Updated operating system components and services
- Verified systems were protected against known vulnerabilities

### 3. Firewall Configuration
- Configured inbound and outbound firewall rules
- Restricted unnecessary network access
- Allowed only approved traffic and services

### 4. File Permissions and Access Control
- Configured file and folder permissions
- Applied the Principle of Least Privilege (PoLP)
- Restricted unauthorized access to sensitive resources

### 5. Removing Unnecessary Devices and Services
- Identified unnecessary services and startup processes
- Disabled unused devices and services
- Reduced the overall attack surface of the systems

### 6. Security Validation and Testing
- Verified security configurations
- Tested firewall rules and access restrictions
- Confirmed system hardening measures were successfully implemented

## Key Security Concepts Applied

### Defense-in-Depth
Implemented multiple layers of security controls including firewalls, permissions, and patch management.

### Principle of Least Privilege (PoLP)
Restricted access rights for users and services to only what was necessary.

### Patch Management
Applied security updates to reduce exposure to publicly known vulnerabilities and exploits.

### Attack Surface Reduction
Disabled unnecessary services and devices to minimize potential attack vectors.
